## ◈ Featured Projects

<details>
<summary><b>⬡ Recall — Production-Grade RAG Pipeline</b></summary>

<br/>

> A production-grade retrieval-augmented generation system built for accuracy, scale, and verifiability. Designed to support multi-format document ingestion, hybrid retrieval, grounded generation with citation verification, and confidence scoring — with an automated evaluation suite to measure every dimension of quality.

<div align="center">

| Attribute | Details |
|-----------|---------|
| **Stack** | Python · FastAPI · OpenAI · ChromaDB · BM25 · Docker · React |
| **Scale** | 50,000+ knowledge records indexed |
| **Retrieval** | Hybrid search: OpenAI embeddings + BM25 + Reciprocal Rank Fusion reranking |
| **Citation Accuracy** | **100%** — citation verification layer eliminates unsupported responses |
| **Evaluation** | 50+ golden Q&A benchmarks measuring correctness, faithfulness, latency, and retrieval quality |
| **Impact** | Automated quality assurance for enterprise knowledge retrieval at scale |
| **Repository** | [![GitHub](https://img.shields.io/badge/View%20Repo-1d4ed8?style=flat-square&logo=github&logoColor=white)](https://github.com/rajverma212/Recall_RAG_Project) |

</div>

Built the hybrid retrieval layer from the ground up: OpenAI embedding search fused with BM25 sparse retrieval, reranked using Reciprocal Rank Fusion to maximize answer relevance. The citation verification layer was engineered to guarantee that every generated response traces directly back to indexed sources — achieving 100% citation accuracy during validation testing and eliminating hallucinated or unsupported answers entirely.

<br/>
</details>

<details>
<summary><b>⬡ Model Regression Detection System (MRDS)</b></summary>

<br/>

> A CI/CD-style AI evaluation platform that detects LLM regressions against versioned datasets before deployment. Built to bring software engineering discipline to AI systems — with automated baselines, deployment quality gates, and a real-time experiment dashboard.

<div align="center">

| Attribute | Details |
|-----------|---------|
| **Stack** | Python · OpenAI · Streamlit · SQLite · Docker · GitHub Actions |
| **Scale** | Versioned dataset evaluation across multiple model checkpoints |
| **Performance** | Automated baseline comparison: accuracy, latency, token usage, cost |
| **Quality Gates** | GitHub Actions blocks releases when AI performance falls below configurable thresholds |
| **Dashboard** | Streamlit interface for experiment tracking, regression analysis, and evaluation reporting |
| **Impact** | Treats AI deployment like software deployment — with rigor and repeatability |
| **Repository** | [![GitHub](https://img.shields.io/badge/View%20Repo-1d4ed8?style=flat-square&logo=github&logoColor=white)](https://github.com/rajverma212/Model_Regression_Detector) |

</div>

Engineered a custom evaluation engine that runs model outputs against versioned golden datasets and computes accuracy, latency, token usage, and cost deltas against the last known-good baseline. Deployment quality gates integrated into GitHub Actions CI/CD pipeline prevent regressions from shipping to production — the same discipline applied to software engineering, applied to AI systems.

<br/>
</details>
